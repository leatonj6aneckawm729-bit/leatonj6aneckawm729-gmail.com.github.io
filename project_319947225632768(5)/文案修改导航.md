# 文案修改导航指南

## 项目概述
- **项目类型**: React + Vite + TypeScript 个人作品集网站
- **主要框架**: React 18.2 + TypeScript
- **UI组件库**: Radix UI + Tailwind CSS
- **主要页面**: 个人信息页面、作品集页面

## 文案修改文件结构

### 1. 主要页面组件
```
src/
├── App.tsx                          # 主应用组件 (页面切换逻辑)
├── components/
│   ├── PersonalInfo.tsx            # 个人信息页面 (主要文案集中地)
│   ├── Portfolio.tsx               # 作品集页面
│   ├── LiteraryWorks.tsx           # 文学作品组件
│   ├── GameDesign.tsx              # 游戏设计组件
│   └── InteractiveVideo.tsx        # 互动视频组件
```

### 2. UI组件库 (包含界面文案)
```
src/components/ui/
├── button.tsx                      # 按钮文案 (如: "提交", "取消")
├── dialog.tsx                      # 对话框文案 (如: "确认", "关闭")
├── alert-dialog.tsx               # 警告对话框文案
├── dropdown-menu.tsx              # 下拉菜单文案
├── form.tsx                       # 表单相关文案
├── input.tsx                      # 输入框placeholder文案
├── textarea.tsx                   # 文本域placeholder文案
├── select.tsx                     # 选择器选项文案
├── checkbox.tsx                   # 复选框文案
├── radio-group.tsx                # 单选按钮文案
├── tabs.tsx                       # 标签页文案
├── tooltip.tsx                    # 提示文案
├── card.tsx                       # 卡片标题和描述文案
├── label.tsx                      # 标签文案
└── navigation-menu.tsx            # 导航菜单文案
```

## 重要文案位置分析

### 🔥 核心文案文件 (优先修改)
1. **PersonalInfo.tsx** - 个人介绍、职业信息、联系方式等核心文案
2. **Portfolio.tsx** - 作品集相关的描述、标题、分类等文案
3. **LiteraryWorks.tsx** - 文学作品的标题、描述、简介等文案
4. **GameDesign.tsx** - 游戏设计相关的介绍、规则说明等文案
5. **InteractiveVideo.tsx** - 互动视频的说明、操作指引等文案

### 🎨 UI界面文案 (次要修改)
1. **form.tsx** - 表单验证消息、提交按钮文案
2. **dialog.tsx** - 弹窗标题、确认/取消按钮文案
3. **button.tsx** - 通用按钮文案
4. **navigation-menu.tsx** - 导航菜单项文案

## 修改建议顺序

### 第一阶段：核心内容文案
1. 先修改 `PersonalInfo.tsx` 中的个人信息和介绍
2. 修改 `Portfolio.tsx` 中的作品集描述
3. 修改各个专项组件 (`LiteraryWorks.tsx`, `GameDesign.tsx`, `InteractiveVideo.tsx`) 的专业内容

### 第二阶段：界面交互文案
1. 修改表单相关的验证和提示文案
2. 修改按钮、对话框等交互元素的文案
3. 修改导航和菜单项的文案

## 修改注意事项

### ✅ 推荐做法
- 保持文案风格统一 (专业、简洁、易读)
- 注意React组件的JSX语法规则
- 保持TypeScript类型检查通过
- 测试文案显示效果和响应式布局

### ⚠️ 注意事项
- 不要修改组件的props接口定义
- 保持原有的CSS类名和样式结构
- 注意硬编码文案和动态文案的区别
- 修改后测试页面功能是否正常

### 🛠️ 修改方法
```typescript
// 示例：修改PersonalInfo.tsx中的文案
const name = "你的新名字";           // 修改姓名
const title = "你的新职位";         // 修改职位
const bio = "你的新个人简介";       // 修改简介
```

## 验证方法
修改完成后，可通过以下方式验证：
1. 检查TypeScript编译无错误
2. 查看页面文案显示效果
3. 测试所有交互功能是否正常
4. 确认响应式布局在不同屏幕尺寸下的表现
