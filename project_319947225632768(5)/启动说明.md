# 🚀 项目启动指南

## ⚠️ 重要提示
**不要直接打开根目录下的 `index.html`**，这会导致 CORS 错误。

## ✅ 正确的打开方式

### 方法 1：使用构建后的文件（推荐）
直接打开 `dist/index.html` 文件，这个版本已经修复了路径问题。

### 方法 2：使用本地服务器
在项目根目录下运行以下命令之一：

#### 使用 Python（推荐）
```bash
python -m http.server 8000 --directory dist
```
然后访问：http://localhost:8000

#### 使用 Node.js serve
```bash
npx serve dist
```

#### 使用 PHP
```bash
php -S localhost:8000 -t dist
```

### 方法 3：使用 VS Code
1. 安装 "Live Server" 插件
2. 右键点击 `dist/index.html`
3. 选择 "Open with Live Server"

## 🐛 故障排除

### 如果看到 CORS 错误：
- 确保你打开的是 `dist/index.html`，而不是根目录的 `index.html`
- 或者使用本地服务器方法

### 如果页面空白：
- 检查浏览器控制台是否有错误信息
- 确保所有文件都在 `dist` 文件夹中

### 如果样式丢失：
- 确保 CSS 文件路径正确
- 检查网络请求是否成功加载资源

## 📁 项目结构
```
project/
├── dist/                 # 构建输出目录 ✅ 使用这个
│   ├── index.html        # 正确的入口文件
│   └── assets/           # 静态资源
├── src/                  # 源代码
├── index.html            # 开发版本（有 CORS 限制）
└── README.md             # 本说明文件
```

## 🎯 为什么要这样做？
现代浏览器出于安全考虑，不允许 `file://` 协议加载 ES 模块（JavaScript 模块）。这是网页安全的标准策略，防止恶意代码访问本地文件。

使用本地服务器可以模拟真实的网页环境，确保应用正常运行。
